System architecture

This system operates as a team of 10 AI agents communicating over an in-process message bus. The orchestrator plans each analysis step using an LLM (Gemma 3 27B), delegating work to specialist agents who invoke skills from a registry of 33 capabilities. Every decision and communication is logged for full transparency.

How agents communicate

Architecture

All agents communicate through a central message bus. The orchestrator delegates tasks and collects results; agents do not coordinate peer-to-peer.

This hub-and-spoke pattern improves debuggability and reproducibility because each decision is captured in the audit trail.
